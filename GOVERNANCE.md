# KubeSynapse Governance

## Overview

KubeSynapse is an open-source project committed to building a production-grade Kubernetes AI platform. This document defines how decisions are made, how contributors can participate, and how the project is maintained.

## Decision Making

### Consensus-Based

Most decisions are made through lazy consensus:
1. A proposal is raised as a GitHub issue or discussion
2. After 7 days with no objections, consensus is reached
3. If objections exist, the maintainers mediate and decide

### Maintainer Votes

For significant changes (API breakage, architectural shifts, security policy):
- A majority of maintainers must approve
- Vetoes require an alternative proposal within 14 days

## Roles

### Contributors

Anyone who submits issues, PRs, or documentation improvements.

### Committers

Contributors with merge rights, appointed by maintainers based on:
- Sustained contribution quality (≥5 merged PRs)
- Understanding of project values and architecture
- Community engagement (reviews, issue triage)

### Maintainers

Project leaders responsible for roadmap, releases, and conflict resolution.
Current maintainers:
- @ykbytes (Project Lead)
