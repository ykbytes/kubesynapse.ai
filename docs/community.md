# KubeSynapse Community

Welcome to the KubeSynapse community! We're building the production-grade platform for Kubernetes-native AI agents — and we'd love your help.

## Get Involved

### Collaboration Channels

- **GitHub Issues:** [github.com/ykbytes/kubesynapse.ai/issues](https://github.com/ykbytes/kubesynapse.ai/issues) — bug reports, feature requests, and implementation follow-up
- **Pull Requests:** [github.com/ykbytes/kubesynapse.ai/pulls](https://github.com/ykbytes/kubesynapse.ai/pulls) — propose code, docs, and chart improvements
- **Email:** `team@kubesynapse.ai` — private inquiries and maintainer contact

- **Monthly Community Meeting:** First Wednesday of every month, 10:00 AM PT / 1:00 PM ET / 19:00 CET

Meeting details will be published in the repository once a recurring schedule is established.

## Contributor Path

### Level 1: Contributor
Anyone who submits issues, comments on discussions, or opens PRs.

**Get started:**
1. Read [CONTRIBUTING.md](../CONTRIBUTING.md)
2. Find a [Good First Issue](https://github.com/ykbytes/kubesynapse.ai/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)
3. Open an issue or pull request and introduce the change you want to make

### Level 2: Committer
Contributors with merge rights. Appointed after:
- ≥ 5 merged PRs
- Consistent code quality and test coverage
- Active participation in code reviews
- Understanding of project architecture

Committers can:
- Merge approved PRs
- Triage issues
- Review and approve community contributions

### Level 3: Maintainer
Project leaders responsible for roadmap, releases, and architectural decisions.

**Current maintainers:**
- @ykbytes — Project Lead

See [GOVERNANCE.md](../GOVERNANCE.md) for full governance details.

### Level 4: Technical Oversight Committee (TOC)
Top-level decision making for major project direction, CNCF sandbox representation, and community standards.

**Current TOC seats:** Open — nominations welcome

## Good First Issues

We tag issues as `good first issue` when they are:
- Self-contained (≤ 100 lines of code)
- Well-defined with clear acceptance criteria
- Not time-sensitive
- Come with mentoring from a maintainer

Look for the label on [GitHub Issues](https://github.com/ykbytes/kubesynapse.ai/issues). If you see one you'd like to work on, leave a comment with your plan so maintainers can coordinate.
